## Changes made

### Brrrdle implementation plan

Added `change-logs/change-log-2026-05-24/implementation-plan.md` with the approved implementation plan for Brrrdle artifact generation and Hugging Face publishing. The plan states that generated Brrrdle artifacts should be transient build outputs, remote Hugging Face destinations should be `latest/brrrdle/` and `data/brrrdle/`, and committed repository folders named `latest/brrrdle/` or `data/brrrdle/` should not be introduced.

### Brrrdle artifact generator

Added `scripts/generate_brrrdle_artifacts.py`, a CLI and helper module that generates Brrrdle-compatible artifacts from the English OpenList valid words file.

The generator:

- Filters input words to sorted, unique, lowercase ASCII alphabetic words of the configured length.
- Defaults to five-letter words.
- Writes `brrrdle_words.txt`.
- Writes `brrrdle_words.json` with metadata and the generated word list.
- Writes `manifest.json` with dataset, release date, source file, word length, word count, and file metadata.
- Writes a generated `README.md` explaining the Brrrdle artifact folder.
- Supports CLI arguments for input path, output directory, release date, and word length.
- Defaults the generated output directory to the current release directory plus `brrrdle/` when no output directory is provided.

### Daily workflow integration

Modified `.github/workflows/daily_update.yml` to add a `Generate Brrrdle artifacts` step after daily release files are committed and before the workflow uploads artifacts and pushes data to Hugging Face.

The new workflow step runs:

```bash
python scripts/generate_brrrdle_artifacts.py \
  --output-dir output/${{ steps.date.outputs.date }}/brrrdle \
  --date ${{ steps.date.outputs.date }}
```

This makes the Brrrdle artifacts available inside the dated release output before the existing artifact upload and Hugging Face upload steps run.

### Hugging Face upload support

Modified `scripts/push_to_huggingface.py` to support uploading generated Brrrdle artifact folders.

The upload changes:

- Added `BRRRDLE_ARTIFACTS_DIRNAME = "brrrdle"`.
- Added `BRRRDLE_REMOTE_PATHS = ("latest/brrrdle", "data/brrrdle")`.
- Added `HuggingFaceUploader.upload_brrrdle_artifacts()`.
- Uploads the generated local Brrrdle artifact folder to both remote Hugging Face dataset paths.
- Returns failure if the Brrrdle artifact directory is missing.
- Authenticates and ensures the Hugging Face dataset repository exists before uploading.
- Added a `--skip-brrrdle` CLI flag.
- Calls the Brrrdle upload during the normal release upload flow unless `--only-full` or `--skip-brrrdle` is used.
- Prints success or failure messages for the Brrrdle upload.
