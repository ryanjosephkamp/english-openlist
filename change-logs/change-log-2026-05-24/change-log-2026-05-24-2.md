## Changes made

### Brrrdle artifact generator

Updated `scripts/generate_brrrdle_artifacts.py` from a single-length, length-5 generator into a multi-length Brrrdle artifact generator.

The generator now:

- Defines the supported Brrrdle length range as 2 through 35.
- Uses `schema_version` value `2.0` for generated metadata.
- Keeps length 5 as the legacy compatibility length.
- Provides helper functions for supported lengths and `words_length_{N}.json` filenames.
- Normalizes input words into a dictionary keyed by word length instead of a single list.
- Preserves the previous lowercase ASCII alphabetic filtering rules.
- Ignores words shorter than 2 letters or longer than 35 letters.
- Loads optional dictionary metadata from `VALID_DICT_FILE`.
- Builds primary word entries with a required `word` field.
- Adds `definition` only when top-level dictionary metadata provides a non-empty string definition.
- Writes primary JSON files named `words_length_2.json` through `words_length_35.json`.
- Writes each primary file as a JSON array of word objects.
- Updates `manifest.json` with:
  - `schema_version`
  - `dictionary_source_file`
  - supported minimum and maximum word lengths
  - total word count across all generated lengths
  - per-length counts
  - the full primary file list
  - transitional legacy compatibility file names
  - the full generated file list
- Keeps transitional legacy length-5 outputs:
  - `brrrdle_words.txt`
  - `brrrdle_words.json`
- Updates the generated Brrrdle `README.md` content to describe primary length-specific files, optional definitions, and transitional compatibility files.
- Removes the CLI `--word-length` option because full 2-through-35 generation is now the default behavior.
- Adds a CLI `--dictionary` option for dictionary metadata input.
- Prints the generated total Brrrdle word count instead of the former single-length word count.

### Daily workflow verification

Updated `.github/workflows/daily_update.yml` to add a `Verify Brrrdle artifacts` step immediately after Brrrdle artifact generation.

The verification step:

- Checks `output/${{ steps.date.outputs.date }}/brrrdle/`.
- Confirms every expected primary file from `words_length_2.json` through `words_length_35.json` exists.
- Fails the workflow with a list of missing files if any primary length artifact is absent.
- Prints a success message after verifying all 34 length artifacts.

The existing generation command and artifact upload flow remain otherwise unchanged.

### Repository README documentation

Updated `README.md` to document daily Brrrdle artifact generation.

The README now states that:

- Daily automation generates transient Brrrdle artifacts in `output/YYYY-MM-DD/brrrdle/`.
- Generated Brrrdle artifacts are uploaded to both `latest/brrrdle/` and `data/brrrdle/` on Hugging Face.
- Generated Brrrdle artifacts are not committed as repository data folders.
- `words_length_{N}.json` files are the primary Brrrdle artifacts.
- Primary files cover word lengths 2 through 35.
- Each primary file is a JSON array of objects with a `word` field.
- `definition` is included only when a non-empty top-level definition exists in `merged_valid_dict.json`.
- `brrrdle_words.txt` and `brrrdle_words.json` remain as length-5 compatibility files for one transition period.
- The legacy length-5 compatibility files should be removed in the next major Brrrdle artifact update.
